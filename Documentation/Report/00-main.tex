\begin{abstract}
We developed a binary hacking challenge in the form of a fake ticketing system that uses a dynamic QR-code which changes every 15 seconds. The goal of the challenge is to extract the ticket out of it enabling it to be stored and used for authentication outside of the provided client application.
This challenge was inspired by a hacker named Conduition who reverse engineered Ticketmaster's SafeTix system in a similar manner. Our application simulates SafeTix, closely mirroring its structure and vulnerabilities, with the main difference being that it is a binary application that needs very different tooling to be reverse engineered. 
Our application consists of both server- and client-side components and can be executed and solved by anyone locally on their computer.

\end{abstract}
